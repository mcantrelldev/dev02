Correctly sizing images on the web is important to the structure of any webpage.
There are several rules to follow when sizing images for use on a web page.

  • Find the right image for the right format
  • Save the image in the format it was created in if saving it locally
  • Save the image at the right size that you will display it as
  • Measure the image using photo editing software and resize it accordingly
  • Name the image so that the name contains helpful info for design

When dealing with different types of content in images, you need to use different types of images.
This is to preserve the colors and content more correctly

A JPG for example is used with actual photos that require many different colors.
A PNG is used for simpler images that use fewer colors like logos.
A GIF is used in similar cases as a PNG and when an image needs to be animated.

Resizing images on a page is mainly tied to resizing the pixels in a way that doesn't distort the image.
An easy way is to divide the current resolution by two, this will preserve the resolution much better than just
resizing it randomly. You should also declare the actual width and height in the html of your page, so it loads faster

Great naming practices include naming your image in with examples of camel case, snake case and dash case
It is also extremely helpful to include the resolution of the image in the name.
Let's break down an image name I used in this webpage;

outsideImage-SM-120x75.jpg

The outsideImage part is a short description of the image, the SM part denotes its size, the 120x75 part denotes the resolution
